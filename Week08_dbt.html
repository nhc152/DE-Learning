W8.03 · Models: view / table / incremental / ephemeral
Chọn materialization đúng để tối ưu runtime và chi phí
📋
1. 📋 Menu Tổng Thể▼
BướcKết quả cần đạtThời gian ước tính
B1Nắm trade-off 4 materializations20 phút
B2Viết incremental model chuẩn25 phút
B3Kiểm tra idempotency sau 2 lần run15 phút
B4So sánh cú pháp PostgreSQL vs Oracle15 phút
🎯
2. 🎯 Mục Đích Module▼
  • Hiểu rõ khi nào dùng view/table/incremental/ephemeral.
  • Viết incremental model với is_incremental() và unique_key.
  • Biết adaptation SQL giữa PostgreSQL và Oracle cho watermark/filter.
  • Xác minh idempotency trước khi áp dụng production schedule.
  • Tránh duplicate và missing data do logic delta sai.

Checklist incremental trước khi merge
1) Có unique_key phản ánh đúng business key
2) Có watermark filter trong is_incremental()
3) Chạy model 2 lần với cùng dữ liệu
4) Query kiểm tra duplicate theo business key
5) Chỉ full-refresh khi có thay đổi logic lớn
fct_host_activity_monthly incremental (map từ homework fact modeling)
{{ config(materialized='incremental', unique_key=['month','host']) }}

with src as (
  select
    {{ dbt.date_trunc('month', 'event_time') }} as month,
    host,
    count(*) as hit_count,
    count(distinct user_id) as unique_visitors,
    max(event_time) as max_event_time
  from {{ source('raw', 'events') }}
  where host is not null
  {% if is_incremental() %}
    and event_time > (select coalesce(max(max_event_time), timestamp '1900-01-01') from {{ this }})
  {% endif %}
  group by 1,2
)
select * from src
Watermark filter: PostgreSQL vs Oracle
🐘 PostgreSQL
🔴 Oracle SQL
PostgreSQL 
where event_time > (
  select coalesce(max(max_event_time), timestamp '1900-01-01 00:00:00')
  from {{ this }}
)
Oracle SQL 
where event_time > (
  select nvl(max(max_event_time), timestamp '1900-01-01 00:00:00')
  from {{ this }}
)
Validation query sau 2 lần run
🐘 PostgreSQL
🔴 Oracle SQL
PostgreSQL 
select month, host, count(*) as dup_cnt
from mart.fct_host_activity_monthly
group by 1,2
having count(*) > 1;
Oracle SQL 
select month, host, count(*) as dup_cnt
from MART.FCT_HOST_ACTIVITY_MONTHLY
group by month, host
having count(*) > 1;
📌 Pattern này bám trực tiếp bài tập fact modeling trong handbook: build bảng reduced/incremental theo thời gian thay vì full rebuild mỗi lần chạy.
📌 Trong model trung tâm dùng cú pháp trung tính adapter; phần khác biệt PostgreSQL/Oracle đặt ở block tab riêng để dễ bảo trì.
⚠️ Nếu query duplicate theo (month, host) trả về kết quả, logic incremental của bạn đang không idempotent.

W8.04 · ref() và source(): Dependency Management
Thiết kế lineage và dependency rõ ràng từ raw đến marts
📋
1. 📋 Menu Tổng Thể▼
BướcKết quả cần đạtThời gian ước tính
B1Khai báo sources trong schema.yml15 phút
B2Refactor hard-code sang ref()/source()25 phút
B3Hiểu selector +model / model+15 phút
B4Đọc lineage và impact khi đổi model15 phút
🎯
2. 🎯 Mục Đích Module▼
  • Dùng source() cho bảng raw có metadata/freshness.
  • Dùng ref() cho dependency giữa các model nội bộ.
  • Loại bỏ hard-code schema.table để tăng tính linh hoạt môi trường.
  • Hiểu tác động upstream/downstream khi model thay đổi.
  • Tăng khả năng debug và deploy theo DAG chuẩn.
📖
3. 📖 Giải Thích Chi Tiết▼

source() là điểm vào của dữ liệu raw; ref() là chuẩn tham chiếu model nội bộ. Khi dùng đúng, dbt dựng được lineage đầy đủ và đảm bảo thứ tự build hợp lý.

schema.yml cho sources + freshness (events/player_seasons)
version: 2

sources:
  - name: raw
    schema: raw
    tables:
      - name: events
        loaded_at_field: event_time
        freshness:
          warn_after: {count: 6, period: hour}
          error_after: {count: 24, period: hour}
      - name: player_seasons
        loaded_at_field: game_date
        freshness:
          warn_after: {count: 1, period: day}
          error_after: {count: 2, period: day}
source() + ref() theo flow staging → marts
-- models/staging/stg_player_seasons.sql
select player_name, season, pts, ast, reb
from {{ source('raw', 'player_seasons') }}

-- models/intermediate/int_players_current.sql
select
  player_name,
  max(season) as current_season,
  avg(pts) as avg_pts_latest_window
from {{ ref('stg_player_seasons') }}
group by 1

-- models/marts/dim_players_scd2.sql
select *
from {{ ref('int_players_current') }}
Dependency selector + freshness
$ dbt ls --select +dim_players_scd2
$ dbt run --select stg_player_seasons+
$ dbt source freshness --select source:raw.player_seasons
Hiển thị upstream/downstream và độ trễ dữ liệu nguồn
✅ Không hard-code schema.table trong model. Dùng ref/source để deploy an toàn giữa dev, staging, prod.

Built-in tests trong schema.yml (SCD + fact reduced)
version: 2

models:
  - name: dim_players_scd2
    columns:
      - name: player_name
        tests: [not_null]
      - name: dbt_scd_id
        tests: [unique, not_null]
      - name: scoring_class
        tests:
          - accepted_values:
              values: ['bad','average','good','star']

  - name: fct_host_activity_monthly
    columns:
      - name: host
        tests:
          - not_null
          - relationships:
              to: ref('stg_events')
              field: host
      - name: month
        tests: [not_null]
Custom generic test: SCD overlap guard
{% test no_scd_overlap(model, business_key, valid_from, valid_to) %}
with ranges as (
  select
    {{ business_key }} as bk,
    {{ valid_from }} as valid_from,
    coalesce({{ valid_to }}, timestamp '9999-12-31 00:00:00') as valid_to,
    dbt_scd_id
  from {{ model }}
)
select a.*
from ranges a
join ranges b
  on a.bk = b.bk
 and a.valid_from < b.valid_to
 and b.valid_from < a.valid_to
 and a.dbt_scd_id <> b.dbt_scd_id
{% endtest %}
Singular test: consistency reduced fact
select *
from {{ ref('fct_host_activity_monthly') }}
where hit_count < unique_visitors
   or hit_count < 0
   or unique_visitors < 0

{% macro scoring_class_from_pts(pts_col) %}
  case
    when {{ pts_col }} > 20 then 'star'
    when {{ pts_col }} > 15 then 'good'
    when {{ pts_col }} > 10 then 'average'
    else 'bad'
  end
{% endmacro %}
macros/date_diff_days.sql
{% macro date_diff_days(start_col, end_col) %}
  {% if target.type == 'postgres' %}
    ({{ end_col }}::date - {{ start_col }}::date)
  {% elif target.type == 'oracle' %}
    trunc({{ end_col }}) - trunc({{ start_col }})
  {% else %}
    datediff(day, {{ start_col }}, {{ end_col }})
  {% endif %}
{% endmacro %}
Dùng macro trong model
select
  player_name,
  avg(pts) as avg_pts,
  {{ scoring_class_from_pts('avg(pts)') }} as scoring_class,
  {{ date_diff_days('min(game_date)', 'max(game_date)') }} as active_span_days
from {{ ref('stg_player_seasons') }}
group by 1
