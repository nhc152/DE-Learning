# output-style.md

## Vietnamese Output Style

### Tone
- calm
- confident
- mentor-like
- practical
- encouraging but not overly emotional

### Sentence Style
Prefer:
- clear sentences
- moderate length
- explicit logic
- concrete wording
- practical phrasing

Avoid:
- overusing buzzwords
- overly long paragraphs without structure
- unnecessary English when Vietnamese is natural
- robotic phrasing

---

## Technical Language Rules
Use Vietnamese explanation with English technical terms where standard.

Examples:
- "Dùng `pandas` để đọc CSV và chuẩn hóa dữ liệu"
- "Kết nối PostgreSQL bằng `sqlalchemy`"
- "So với PostgreSQL, Oracle SQL khác ở cú pháp pagination"

Keep code exactly in its native syntax.

---

## Section Writing Expectations

### Menu Tổng Thể
- short overview
- quick orientation
- useful learning path summary

### Mục Đích Module
- clear bullets
- outcome-based
- explain what learner will be able to do

### Giải Thích Chi Tiết
- practical explanation first
- then examples
- then syntax or code
- then notes / warnings / tips if helpful

### Câu Hỏi Ôn Tập
- application-oriented
- not trivial
- explanations should teach, not just mark right/wrong

### Tổng Kết
- concise
- structured
- memorable
- recap the most important ideas

### Bài Tập Thực Hành
- realistic
- achievable
- connected to the lesson
- solutions should be instructive, not just short answers

---

## SQL Presentation Style
If showing both PostgreSQL and Oracle SQL:
- label clearly
- separate code blocks if syntax differs
- explain the difference in one short Vietnamese note

Example:
- PostgreSQL dùng `LIMIT`
- Oracle thường dùng `FETCH FIRST ... ROWS ONLY`

---
## UI consistency
Use database toggle tabs for all SQL comparisons.
Preferred style:
[PostgreSQL] [Oracle SQL]
Switch visible code block with JavaScript.

## UX Writing Style
Button labels, headings, and section names should be Vietnamese and user-friendly.

Examples:
- "Xem lời giải"
- "Module tiếp theo"
- "Làm lại từ đầu"
- "Chấm điểm mini test"

---

## Consistency Rule
Every week should feel like it belongs to the same tutorial family as Week00.
