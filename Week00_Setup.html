2. Tại Sao Setup Quan Trọng? ▼

Nhiều người bỏ qua tuần setup vì nghĩ "cài mấy cái tool đơn giản, làm nhanh thôi". Đây là sai lầm phổ biến nhất dẫn đến hàng giờ debug vô nghĩa về sau.

🔴 Hậu quả của setup sai

  • Docker version không tương thích → docker compose up lỗi cú pháp kỳ lạ
  • Python wrong version (3.8 vs 3.11) → thư viện mới không cài được, type hints không hoạt động
  • Thiếu Java JDK → PySpark khởi động thất bại với lỗi JAVA_HOME not set
  • PostgreSQL port conflict → container không start được, mất 1 tiếng tìm lỗi
  • Git SSH không cấu hình → mỗi lần push phải nhập password
✅ Nguyên tắc vàng: Đừng bỏ qua tuần này — setup sai là nguồn gốc của 80% lỗi kỳ lạ về sau. Dành 3 giờ setup đúng ngay từ đầu, tiết kiệm 30 giờ debug sau này.

3. Giải Thích Chi Tiết — Tại Sao Từng Tool? ▼

🐳 Docker — Tại sao không cài native?

Cài PostgreSQL native trực tiếp trên máy gây ra nhiều vấn đề: conflict version, khó xóa sạch, không portable. Docker cho phép chạy bất kỳ service nào trong container cô lập — xóa sạch chỉ với 1 lệnh, team khác clone repo và chạy y hệt.

🐘 PostgreSQL — Tại sao không dùng SQLite?

SQLite không hỗ trợ: window functions đầy đủ, composite types (film_info[]), concurrent connections, ENUM types. Đây là những tính năng cốt lõi trong data modeling tuần 5–7.

🐍 Python 3.11+ — Tại sao không 3.8/3.9?

Python 3.11 nhanh hơn 3.10 khoảng 10–60%. dbt 1.6+, PySpark 3.4+, và các thư viện DE mới nhất yêu cầu 3.10 minimum. Dùng 3.11 đảm bảo tương thích tất cả giai đoạn sau.

⚡ Spark — Tại sao cài từ tuần 0?

PySpark cần thêm Java JDK — nếu để đến tuần 16 mới cài thì dễ bỏ sót. Cài sớm, test sớm, phát hiện vấn đề Java/RAM sớm hơn.

📝 Git/GitHub — Tại sao phải commit từ ngày đầu?

Portfolio DE được đánh giá qua commit history. Recruiter nhìn vào graph — nếu thấy commit liên tục 6 tháng đó là bằng chứng thực tế của sự kiên trì. Bắt đầu commit từ tuần 0 = 28 tuần commit history.

      POSTGRES_DB: postgres

    container_name: de-pgadmin

      PGADMIN_DEFAULT_PASSWORD: admin
