Phân vai Python và SQL trong pipeline

  • SQL: xử lý set-based (join, aggregate, merge) ngay trong database để tận dụng engine.
  • Python: điều phối luồng, gọi API/file, chuẩn hóa dữ liệu, kiểm tra chất lượng và kích hoạt câu SQL đúng thời điểm.

Exit mindset sau module W2.01

  • Idempotent: chạy lại cùng input không tạo duplicate, không làm sai số liệu.
  • Observable: mỗi bước ETL đều có log đủ ngữ cảnh để truy vết sự cố.
  • Recoverable: khi lỗi tạm thời xảy ra, pipeline có retry giới hạn thay vì fail ngầm.

⚠️ Anti-pattern phổ biến: gom toàn bộ pipeline vào một file dài, không log, không chia step. Khi lỗi xảy ra sẽ rất khó biết hỏng ở đâu.

Upsert: PostgreSQL vs Oracle SQL

sql/postgres_upsert.sql
-- PostgreSQL
INSERT INTO dim_customer (customer_id, city, updated_at)
VALUES (:customer_id, :city, NOW())
ON CONFLICT (customer_id)
DO UPDATE SET
  city = EXCLUDED.city,
  updated_at = NOW();
sql/oracle_merge.sql
-- Oracle SQL
MERGE INTO dim_customer d
USING (
  SELECT :customer_id customer_id, :city city FROM dual
) s
ON (d.customer_id = s.customer_id)
WHEN MATCHED THEN UPDATE SET
  d.city = s.city,
  d.updated_at = SYSTIMESTAMP
WHEN NOT MATCHED THEN INSERT (customer_id, city, updated_at)
VALUES (s.customer_id, s.city, SYSTIMESTAMP);
📌 PostgreSQL tối ưu với ON CONFLICT; Oracle SQL trong hệ thống doanh nghiệp thường dùng MERGE để đạt cùng mục tiêu upsert.

Offset vs Cursor pagination: chọn đúng pattern theo API

python/pagination_patterns.py
# Offset pagination: page=1,2,3...
def fetch_offset(session, base_url, token):
    page, all_rows = 1, []
    while True:
        resp = session.get(
            f"{base_url}/orders",
            headers={"Authorization": f"Bearer {token}"},
            params={"page": page, "page_size": 500},
            timeout=30
        )
        resp.raise_for_status()
        payload = resp.json()
        rows = payload.get("data", [])
        if not rows:
            break
        all_rows.extend(rows)
        page += 1
    return all_rows

# Cursor pagination: dùng next_page_token/next_cursor
def fetch_cursor(session, base_url, token):
    token_cursor, all_rows = None, []
    while True:
        params = {"limit": 500}
        if token_cursor:
            params["cursor"] = token_cursor
        resp = session.get(
            f"{base_url}/orders",
            headers={"Authorization": f"Bearer {token}"},
            params=params,
            timeout=30
        )
        resp.raise_for_status()
        payload = resp.json()
        all_rows.extend(payload.get("data", []))
        token_cursor = payload.get("next_page_token")
        if not token_cursor:
            break
    return all_rows

Retry có exponential backoff + jitter cho 429/5xx

python/request_with_retry.py
import random
import time
import requests

def request_with_retry(session, url, headers, params, max_attempts=5):
    for attempt in range(max_attempts):
        resp = session.get(url, headers=headers, params=params, timeout=30)
        if resp.status_code in [429, 500, 502, 503, 504]:
            base = min(2 ** attempt, 30)
            jitter = random.uniform(0, 1)
            time.sleep(base + jitter)
            continue
        resp.raise_for_status()
        return resp.json()
    raise requests.exceptions.RequestException("Hết số lần retry cho API request")
📌 Vì sao cần jitter? Nếu nhiều job cùng retry đúng một mốc thời gian (2s, 4s, 8s) sẽ gây thundering herd và làm API tiếp tục quá tải. Jitter giúp phân tán thời điểm retry.

import json
import logging
from datetime import datetime

class JSONFormatter(logging.Formatter):
    """Custom formatter ghi log dạng JSON — dễ parse bởi Datadog/Grafana."""
    def format(self, record):
        log_entry = {
            "timestamp": datetime.utcnow().isoformat() + "Z",
            "level": record.levelname,
            "logger": record.name,
            "message": record.getMessage(),
            "module": record.module
        }
        if record.exc_info:
            log_entry["exception"] = self.formatException(record.exc_info)
        return json.dumps(log_entry)

🔁 Retry Decorator tái sử dụng

utils.py · retry decorator
import functools
import time
import logging

logger = logging.getLogger(__name__)

def retry(max_attempts: int = 3, delay: float = 1.0,
          exceptions: tuple = (Exception,)):
    """Decorator retry với exponential backoff."""
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for attempt in range(max_attempts):
                try:
                    return func(*args, **kwargs)
                except exceptions as e:
                    if attempt == max_attempts - 1:
                        logger.error(f"{func.__name__} failed sau {max_attempts} lần: {e}")
                        raise
                    wait = delay * (2 ** attempt)
                    logger.warning(f"Attempt {attempt+1}/{max_attempts} failed. Retry sau {wait}s")
                    time.sleep(wait)
        return wrapper
    return decorator

# Sử dụng:
@retry(max_attempts=3, delay=2.0, exceptions=(requests.exceptions.RequestException,))
def fetch_data_from_api(url: str) -> dict:
    response = requests.get(url, timeout=10)
    response.raise_for_status()
    return response.json()

import logging, sys, os

def setup_logging(level: str = "INFO", log_file: str = None):
    handlers = [logging.StreamHandler(sys.stdout)]
    if log_file:
        os.makedirs(os.path.dirname(log_file), exist_ok=True)
        handlers.append(logging.FileHandler(log_file))

    logging.basicConfig(
        level=getattr(logging, level.upper()),
        format="%(asctime)s | %(levelname)-8s | %(name)s | %(message)s",
        datefmt="%Y-%m-%d %H:%M:%S",
        handlers=handlers
    )

# Test

🏗️ Class-based ETL Pipeline

pipeline.py · full structure
import argparse
import pandas as pd
import logging
from datetime import date
from sqlalchemy import create_engine

from config import Config
from extract import read_csv_safe, fetch_all_orders
from transform import clean_orders, create_daily_summary
from load import incremental_load

logger = logging.getLogger(__name__)

class OrderPipeline:
    """Pipeline ETL cho orders data: API → transform → PostgreSQL."""

    def __init__(self, config: Config):
        self.config = config
        self.engine = create_engine(
            f"postgresql+psycopg2://{config.db_user}:{config.db_password}"
            f"@{config.db_host}:{config.db_port}/{config.db_name}"
        )

    def extract(self, run_date: str) -> pd.DataFrame:
        logger.info(f"[EXTRACT] Bắt đầu cho ngày {run_date}")
        records = fetch_all_orders(self.config, run_date)
        df = pd.DataFrame(records)
        logger.info(f"[EXTRACT] Xong: {len(df)} rows")
        return df

    def transform(self, df: pd.DataFrame) -> tuple[pd.DataFrame, pd.DataFrame]:
        logger.info("[TRANSFORM] Bắt đầu")
        clean = clean_orders(df)
        summary = create_daily_summary(clean)
        logger.info(f"[TRANSFORM] Clean: {len(clean)} rows | Summary: {len(summary)} rows")
        return clean, summary

    def load(self, clean: pd.DataFrame, summary: pd.DataFrame) -> None:
        logger.info("[LOAD] Bắt đầu")
        inserted = incremental_load(clean, self.engine, "orders", "order_id")
        summary.to_sql("daily_summary", self.engine, if_exists="append", index=False)
        logger.info(f"[LOAD] Xong: {inserted} orders mới, {len(summary)} summary rows")

    def run(self, run_date: str) -> None:
        logger.info(f"=== Pipeline bắt đầu: {run_date} ===")
        raw = self.extract(run_date)
        clean, summary = self.transform(raw)
        self.load(clean, summary)
        logger.info(f"=== Pipeline hoàn thành: {run_date} ===")

# Week 2 — Python ETL Pipeline

## Mục tiêu
Pipeline ETL đọc orders từ CSV và REST API, làm sạch data,
tính daily summary, và load vào PostgreSQL incremental.

## Input
- CSV: `data/sample_orders.csv` (từ source system)
- API: JSONPlaceholder (demo endpoint)

## Output
- Table `orders` trong PostgreSQL (incremental insert)
- Table `daily_summary` theo ngày + channel
- Parquet file tại `output/daily_summary.parquet`

## Cách chạy
```bash
pip install -r requirements.txt
cp .env.example .env  # edit với giá trị thật
docker compose up -d  # start PostgreSQL (từ tuần 0)
python pipeline.py --date 2024-01-15
python pipeline.py  # mặc định: hôm nay
```

## Design Decisions
- **Incremental load**: chỉ insert orders mới để idempotent
- **Left join** khi merge orders với users: không bỏ orders dù thiếu user
- **snappy compression**: balance speed/size cho Parquet
- **JSON logging**: dễ parse khi deploy lên production log system
