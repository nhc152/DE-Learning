NỘI DUNG CẦN BAO PHỦ:
- OLTP vs OLAP:
  - row-based vs columnar
  - transactional workload vs analytical workload
  - concurrency pattern khác nhau
  - vì sao PostgreSQL tốt cho app DB nhưng không phải lúc nào cũng tối ưu cho analytics lớn
- DuckDB:
  - local analytics engine
  - query trực tiếp trên Parquet
  - không cần server riêng
  - use case cho data exploration, local benchmarking, analytics notebook
  - hạn chế của DuckDB
- ClickHouse:
  - columnar storage
  - rất mạnh cho event/time-series analytics
  - ingestion nhanh
  - aggregation nhanh
  - primary key / sorting key mindset
  - partitioning basics
  - use cases thật
- Apache Druid:
  - low-latency OLAP
  - real-time ingestion mindset
  - pre-aggregated / serving layer use cases
  - dashboard workloads
  - khi nào Druid phù hợp, khi nào không
- Pre-aggregation / serving:
  - materialized views
  - rollups
  - serving tables
  - latency vs freshness trade-off
  - dashboard optimization mindset
- Benchmark mindset:
  - benchmark phải công bằng
  - cùng dataset, cùng query, cùng mục tiêu
  - scan volume, latency, concurrency
  - không benchmark kiểu cảm tính
- Decision framework:
  - khi nào dùng DuckDB
  - khi nào dùng ClickHouse
  - khi nào warehouse cloud vẫn hợp lý hơn
  - trade-off về ops burden, latency, cost, concurrency
- Pipeline patterns:
  - Kafka → ClickHouse cho real-time event analytics
  - S3/Parquet → DuckDB cho local / ad hoc analysis
  - khi nào cần precompute serving table
- Artifact modern_olap/ gồm:
  - benchmark_queries.sql
  - duckdb_demo.sql
  - clickhouse_schema.sql
  - pipeline_notes.md
  - engine_comparison.md
  - README.md mô tả workload, benchmark setup, trade-offs
