NỘI DUNG CẦN BAO PHỦ:
- Analytical patterns là gì:
  - không chỉ là query ad hoc
  - là các pattern lặp lại phục vụ KPI, dashboard, experimentation, product insight
  - cần data model, grain, event definition rõ ràng
- State change tracking:
  - theo dõi entity đổi trạng thái theo thời gian
  - ví dụ user subscription, order status, ticket lifecycle
  - event table vs current snapshot
  - SQL pattern để detect transition
- GROUPING SETS / ROLLUP / CUBE:
  - phân tích đa chiều
  - subtotal và grand total
  - khi nào dùng thay vì nhiều UNION ALL
  - đọc output đúng cách
- Streak analysis:
  - consecutive active days
  - gap-and-island pattern
  - user engagement use case
  - rolling activity patterns
- Funnel analysis:
  - event steps
  - ordered conversion
  - drop-off per step
  - session-based vs user-based funnel
  - pitfalls khi đếm funnel
- Cohort analysis:
  - cohort theo first activity / signup / first purchase
  - retention day 1 / day 7 / day 30
  - cohort matrix
  - difference giữa retention và reactivation nếu phù hợp
- Attribution modeling:
  - first-touch
  - last-touch
  - multi-touch
  - marketing / product attribution basics
  - giới hạn của mô hình attribution đơn giản
- Production considerations:
  - grain definition
  - event quality
  - deduplication
  - late-arriving events
  - partitioning / indexing / materialized summaries nếu phù hợp
- Artifact analytical_patterns/ gồm:
  - state_change_tracking.sql
  - grouping_sets_multidim.sql
  - streak_analysis.sql
  - funnel_analysis.sql
  - cohort_retention.sql
  - attribution_modeling.sql
  - README.md mô tả use case, assumptions, grain, pitfalls
