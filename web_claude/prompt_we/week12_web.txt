NỘI DUNG CẦN BAO PHỦ:
- Tại sao data quality là vấn đề production, không chỉ là "check thêm cho đẹp"
- Testing pyramid cho DE:
  - unit test cho transform logic
  - integration test cho pipeline
  - data quality test cho output
- Great Expectations:
  - ExpectationSuite
  - Checkpoint
  - data docs
  - workflow thực tế
- Soda:
  - SQL-based checks
  - scan.yml
  - khi nào dùng Soda thay vì Great Expectations
- dbt tests nâng cao:
  - built-in vs custom tests
  - dbt-expectations nếu phù hợp
  - custom macros / test patterns
- Anomaly detection cơ bản:
  - row count deviation
  - null spike
  - value range anomalies
  - freshness anomalies
- Data contracts:
  - schema contract
  - SLA giữa producer và consumer
  - fail-fast mindset
- Artifact data_quality/ gồm:
  - great_expectations/expectations/orders_suite.json
  - great_expectations/checkpoints/daily_check.yml
  - soda/checks/orders_checks.yml
  - dbt_tests/custom_test_accepted_range.sql
  - README.md mô tả test strategy, alerting, fail conditions

YÊU CẦU SQL:
- Ở những phần liên quan đến row count checks, null checks, uniqueness checks, accepted range checks, freshness checks, anomaly validation queries hoặc reconciliation queries, hãy minh họa bằng cả PostgreSQL và Oracle SQL nếu hợp lý
- Nếu cú pháp khác nhau, trình bày rõ riêng từng block
- Nếu project rules yêu cầu SQL tabs thì dùng tab PostgreSQL / Oracle SQL thay vì ghi text thuần
- Giải thích ngắn gọn điểm khác nhau

YÊU CẦU QUIZ:
