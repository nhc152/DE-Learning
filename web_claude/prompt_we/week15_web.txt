NỘI DUNG CẦN BAO PHỦ:
- Vai trò của ingestion trong data platform:
  - source systems → raw layer → staging / warehouse
  - ingestion không chỉ là copy data
  - reliability, freshness, schema stability, ownership
- ELT vs ETL:
  - vì sao ELT thắng thế với cloud warehouse
  - khi nào ETL vẫn hợp lý
  - transform nên đặt ở đâu
- Airbyte:
  - connector-based mindset
  - source / destination / connection
  - sync mode
  - full refresh vs incremental
  - state management basics
  - custom connector concept
  - ưu và nhược điểm
- dlt:
  - Python-native ingestion
  - phù hợp cho API, custom business logic, lightweight loading
  - pipeline pattern
  - load từ API vào warehouse
  - incremental loading concept
  - schema inference / control
- Fivetran:
  - managed ELT
  - ưu điểm về tốc độ triển khai, reliability, maintenance
  - nhược điểm về cost / control
  - khi nào startup / team nhỏ / enterprise nên dùng
- CDC:
  - Change Data Capture là gì
  - log-based CDC vs trigger-based CDC
  - Debezium concept
  - latency / reliability / source DB impact
  - use cases thật
- Schema evolution:
  - thêm cột mới
  - đổi type
  - column missing / renamed
  - pipeline breakage
  - contract mindset cho ingestion
- Decision framework:
  - khi nào dùng Airbyte
  - khi nào dùng dlt
  - khi nào dùng Fivetran
  - trade-off: cost, speed, flexibility, ops burden
- Artifact data_ingestion/ gồm:
  - airbyte/connection_config.json
  - dlt_pipeline/load_api_to_dw.py
  - cdc_demo/debezium_config.json
  - README.md mô tả khi nào dùng Airbyte vs dlt vs Fivetran
  - architecture.md mô tả flow source → raw → warehouse
