NỘI DUNG CẦN BAO PHỦ:
- Vì sao PostgreSQL local không còn đủ khi dữ liệu và workload analytics tăng
- Vì sao warehouse hiện đại dùng columnar storage
- Khác nhau giữa row-based OLTP và columnar analytics engines
- Partitioning, clustering, pruning, materialized view mindset
- So sánh Snowflake / BigQuery / Redshift theo góc nhìn DE:
  - compute/storage separation
  - concurrency
  - cost model
  - operational overhead
  - ecosystem fit
- Schema-on-write vs schema-on-read:
  - khi nào strict schema hữu ích
  - khi nào flexible schema hữu ích
  - rủi ro của mỗi cách
- Lakehouse:
  - vì sao Data Lake thuần không đủ
  - vì sao Warehouse thuần đôi khi đắt / thiếu linh hoạt
  - Lakehouse như lớp dung hòa
- Apache Iceberg:
  - table format là gì
  - metadata layers
  - snapshots
  - time travel
  - schema evolution
  - partition evolution
  - ACID trên object storage
- dbt migration:
  - điều chỉnh source / target profile
  - model materialization khác nhau
  - incremental model trên warehouse
  - source freshness
  - warehouse-specific optimization mindset
- Cost & performance:
  - partition pruning
  - limit scan bytes
  - avoid SELECT *
  - materialized views
  - file sizing / compaction mindset nếu phù hợp
  - warehouse cost trade-off
- Artifact warehouse_migration/ gồm:
  - dbt_project/
  - profiles_example/
  - migration_notes.md
  - cost_comparison.md
  - iceberg_demo.md
  - README.md mô tả kiến trúc, migration steps, cost/performance notes
