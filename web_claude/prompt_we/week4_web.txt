NỘI DUNG CẦN BAO PHỦ:
- Pipeline là gì trong Data Engineering
- DAG: node, edge, dependency, upstream/downstream
- Idempotency: chạy nhiều lần cho cùng kết quả
- Full load vs incremental vs backfill
- Failure handling: retry, dead letter queue, alerting
- Monitoring cơ bản: row count, null check, freshness check
- SLA, SLO, SLI cho data pipeline
- Data contracts giữa producer và consumer
- Failure scenarios thực tế và cách xử lý
- Artifact week4_pipeline_design.md gồm:
  - DAG mô tả pipeline thực tế
  - idempotency strategy
  - failure scenarios + cách xử lý
  - monitoring checklist
  - trade-off full load vs incremental

YÊU CẦU SQL:
- Ở những phần liên quan đến validation, quality checks, watermark checks, duplicate prevention, row count checks, freshness checks hoặc load verification, hãy minh họa bằng cả PostgreSQL và Oracle SQL nếu hợp lý
- Nếu cú pháp khác nhau, trình bày rõ riêng từng block
