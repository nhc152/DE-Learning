PHẠM VI NỘI DUNG TUẦN 10-11:
Tên: Tuần 10-11 – Apache Airflow: Orchestration
File: Week10_Airflow.html

Các module bắt buộc:
- W10.01 – Airflow Architecture (Scheduler/Executor/Webserver)
- W10.02 – DAG: Định Nghĩa & Schedule
- W10.03 – Core Operators (Python/Bash/SQL)
- W10.04 – Task Dependencies & XCom
- W10.05 – Variables, Connections & Secrets
- W10.06 – Sensors & ExternalTaskSensor
- W10.07 – Dynamic DAGs từ Config
- W10.08 – Tích Hợp dbt + Airflow
- W10.09 – Retry, Timeout & SLA Miss
- W10.10 – Artifact: airflow_dags/ hoàn chỉnh
- W10.11 – Mini Test Tổng Kết

NỘI DUNG CẦN BAO PHỦ:
- Kiến trúc Airflow: Scheduler, Executor, Worker, Metadata DB, Webserver
- DAG là gì, schedule bằng cron, @daily, timedelta
- Core Operators: PythonOperator, BashOperator, SQLExecuteQueryOperator
- Task dependencies: >>, <<, set_upstream, set_downstream
- XCom để truyền data giữa tasks
- Variables, Connections, quản lý config và secrets
- Sensors: FileSensor, ExternalTaskSensor, HttpSensor
- Retry, timeout, SLA miss
- Dynamic DAGs generate từ config
- Tích hợp dbt + Airflow: DbtRunOperator, DbtTestOperator hoặc pattern tương đương
- README setup local bằng Docker Compose
- Artifact airflow_dags/ gồm:
  - dags/etl_orders_daily.py
  - dags/dbt_pipeline.py
  - dags/data_quality_check.py
  - dags/backfill_historical.py

YÊU CẦU SQL:
- Ở những phần liên quan đến quality checks, row count validation, freshness checks, metadata checks, load verification hoặc DAG downstream validation, hãy minh họa bằng cả PostgreSQL và Oracle SQL nếu hợp lý
- Nếu cú pháp khác nhau, trình bày rõ riêng từng block
