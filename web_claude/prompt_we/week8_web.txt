NỘI DUNG CẦN BAO PHỦ:
- dbt là gì, vai trò trong ELT, khác ETL truyền thống ra sao
- Project structure: models/, tests/, macros/, seeds/, snapshots/
- Model materializations: view, table, incremental, ephemeral
- ref() và source(): dependency, lineage, môi trường staging → marts
- Built-in tests: unique, not_null, accepted_values, relationships
- Custom tests và dbt-expectations nếu phù hợp
- Documentation: schema.yml, dbt docs generate, dbt docs serve
- Snapshots để implement SCD Type 2
- Macros để DRY SQL logic
- Commands: dbt run, dbt test, dbt build, dbt source freshness
- Artifact dbt_project/ gồm:
  - models/staging/stg_orders.sql
  - models/staging/stg_customers.sql
  - models/intermediate/int_order_items_enriched.sql
  - models/marts/fct_orders.sql
  - models/marts/dim_customers.sql
  - tests/generic/
  - tests/singular/
  - README.md với lineage diagram, cách chạy, test coverage

YÊU CẦU SQL:
- Phải dùng cả PostgreSQL và Oracle SQL ở các phần liên quan
- Nếu cú pháp khác nhau, trình bày rõ riêng từng block
- Tập trung vào:
  - source/staging SQL
  - incremental models
  - snapshot logic
  - validation queries
  - test queries
- Giải thích ngắn gọn điểm khác nhau giữa PostgreSQL và Oracle SQL
- Nếu có chỗ dbt phù hợp hơn với PostgreSQL hơn Oracle, hãy nói rõ limitation hoặc adaptation cần thiết
